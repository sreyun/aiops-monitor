## 1. 构建系统概览
本项目采用多语言、多产物的构建体系：Go（Server + Agent）、Kotlin/Android App、Python 插件，通过 Docker 多阶段镜像统一打包，并由 GitHub Actions 驱动 CI/CD。核心思路是本地用脚本快速编译，CI 用 buildx 做跨架构镜像构建并推送至华为云 SWR。

## 2. 关键文件与职责
- go.mod / go.sum：Go 模块定义（Go 1.22），依赖通过 -mod=vendor 使用 vendor/ 目录。
- cmd/server/main.go、cmd/agent/main.go：两个独立 Go 入口，分别产出 aiops-server 与 aiops-agent。
- docker/Dockerfile：多阶段构建（builder → server/agent 镜像），支持 linux/amd64 + linux/arm64 双架构；同时为一键安装分发预编译所有平台 Agent 二进制。
- docker-compose.yml：编排 aiops-server + VictoriaMetrics + PostgreSQL + 可选 agent，提供开发/演示环境一键启动。
- .github/workflows/release.yml：触发条件为 v* tag 或手动 dispatch；自动登录华为云 SWR、buildx 多架构构建、打版本+latest 标签并推送。
- build.ps1：Windows 本地构建脚本，从 git tag 注入 main.appVersion，支持交叉编译 Linux/macOS。
- deploy.sh：将已编译的 aiops-server.exe 通过 scp/ssh 部署到远程主机并重启服务。
- android/build.gradle：Android Gradle 工程，由 Android Studio 或命令行 gradlew 构建 APK。
- plugins/requirements.txt：Agent 内嵌 Python 插件依赖，随镜像一起安装。
- scripts/build_i18n_*.js：前端 i18n 资源构建脚本（被 Server 内嵌）。

## 3. 架构与约定
### 3.1 版本注入策略
- 统一变量：main.appVersion 通过 -ldflags "-X main.appVersion=${VERSION}" 注入。
- 来源：Docker 构建时由 --build-arg VERSION 传入（CI 中取 git tag）；本地 PowerShell 脚本取 git describe --tags，无 tag 则回退到短 commit hash。
- 效果：/healthz 或 CLI --version 可输出精确版本，便于追踪发布。

### 3.2 多架构与 CGO
- 所有 Go 构建均设置 CGO_ENABLED=0，确保静态链接、零运行时依赖，适合 Alpine 基础镜像。
- Dockerfile 在 builder 阶段以 TARGETOS/TARGETARCH 为目标平台交叉编译，再由 buildx 并行产出多架构 manifest。
- 除运行期镜像外，还额外编译 Windows/macOS/Linux 全平台 Agent 二进制，供 Server 的 /dl/ 下载端点分发。

### 3.3 依赖管理
- Go 依赖使用 vendor/ 目录 + GOFLAGS=-mod=vendor，避免网络拉取，保证离线构建与可重现性。
- Android 使用 Gradle 8.5.2 + Kotlin 1.9.24，仓库源为 google/mavenCentral。
- Python 插件通过 pip3 install -r requirements.txt 在镜像构建期安装。

### 3.4 容器化分层
- server 镜像：仅含 aiops-server 二进制 + 内嵌前端 + plugins.zip + 配置示例，暴露 8529，健康检查 /healthz。
- agent 镜像：包含 aiops-agent + Python 运行环境与插件，默认连接 http://aiops-server:8529。
- dev 镜像：docker/Dockerfile.dev 用于热重载开发（compose 中指定 target）。

### 3.5 CI/CD 流水线
- Release 流水线：push v* 或手动触发 → QEMU + buildx → 构建 server/agent 多架构镜像 → 登录华为云 SWR → 推送 ${VERSION} 与可选 latest 标签。
- CodeQL 安全扫描：每 PR/定时任务对 Go 代码执行数据流/污点分析，结果写入 GitHub Security tab。
- 安全扫描：security.yml 负责依赖漏洞扫描。

## 4. 开发者应遵循的规则
1. 版本号：新增功能后打 vX.Y.Z tag 触发 Release 流水线；本地调试可用 git describe --tags 生成 dev 版本。
2. 新增 Go 依赖：务必 go mod vendor 同步 vendor 目录，否则 CI 会因无法拉包失败。
3. 修改 Dockerfile：如需新增目标平台，需同步更新 release.yml 的 PLATFORMS 环境变量及 Dockerfile 中对应 GOOS/GOARCH 构建块。
4. Android 改动：通过 Android Studio 或 ./gradlew assembleDebug 构建，不要直接提交 APK 到仓库。
5. 插件扩展：新增 Python 插件需在 plugins/requirements.txt 添加依赖，并在 docker/Dockerfile 的 agent 阶段确认会被复制进镜像。
6. 本地快速构建：Windows 下使用 powershell -File build.ps1 [-CrossCompile]；Linux/macOS 可直接 go build ./cmd/server 与 ./cmd/agent。
7. 生产部署：优先使用 docker compose 或 SWR 预构建镜像；裸机部署参考 deploy.sh，注意修改 DSN、TLS、密钥等敏感环境变量。
