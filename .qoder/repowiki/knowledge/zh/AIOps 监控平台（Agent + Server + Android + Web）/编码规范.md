- 所有 HTTP 路由集中在 `handlers.go` 的 `Routes()` 方法中，使用 Go 1.22 的 `mux.HandleFunc("METHOD /path/{param}", handler)` 模式注册，避免分散的 `http.HandleFunc` 调用。
- 跨进程通信的数据结构统一定义在 `shared/wire.go` 中，由 agent 与 server 共同 import，禁止各自重复声明相同字段，保证 wire contract 不漂移。
- 新增采集器遵循「独立文件 + 配置结构体 + reporter 回调」模式：在 `cmd/agent/collector_xxx.go` 中定义 `Config` 结构体并在 `config.json` 中启用，采集结果通过 `reporter(shared.XxxReport{...})` 推送，而非直接写全局变量。
- 平台差异通过 Go build tag 拆分文件（如 `collector_linux.go` / `collector_windows.go` / `pty_unix.go` / `pty_windows.go`），每个平台只实现自身逻辑，`collector_other.go` 提供空实现兜底。
- HTTP 响应统一经 `writeJSON(w, status, v)` 写入，且通过 `gzipMiddleware` + `gzipWriterPool` 对非 WebSocket/代理路径自动压缩，避免每请求分配 gzip.Writer。
- 敏感凭据优先从环境变量读取（如 `RedfishTarget.PasswordEnv`），仅在不可用时回退到配置文件中的明文字段，并在日志中给出明确修复提示。
