仓库采用「双二进制 + 共享协议」的多组件架构：
- `cmd/server` 是主服务入口，通过 `main.go` 启动 HTTP 服务器并挂载 `handlers.go` 中用 Go 1.22 method+path pattern 注册的 `/api/v1/*` REST 路由；HTTP 中间件链为 `securityHeaders → cors → gzip → bodyLimit → auth`，静态资源通过 `//go:embed web` 内嵌，Dashboard JS 模块在运行时拼接为单个 `/app.js`。
- 存储层统一依赖 PostgreSQL（关系型数据）+ VictoriaMetrics（时序数据），由环境变量 `AIOPS_POSTGRES_DSN` / `AIOPS_VM_URL` 驱动，内置 aiops.db 已废弃；`mustOpenPG` 带重试以配合 docker-compose 冷启动。
- `cmd/agent` 是部署到目标主机的轻量 Agent，`main.go` 解析配置后创建 `Agent`，按周期调用 `Collector.Collect()` 上报基础指标，并通过插件系统执行 Python 脚本；同时可选启动三类增强采集器：`collector_redfish.go`（轮询 BMC Redfish API 拉取整机/RAID/电源/温度/事件）、`collector_netflow.go`（UDP 监听 NetFlow v5/v9 并做五元组窗口聚合）、`collector_packet.go`（Linux 下 diff `/proc/net/nf_conntrack` 生成增量流记录），三者均通过 `reporter` 回调把 `HardwareReport` / `NetFlowReport` 推给服务端。
- `shared/wire.go` 定义 Agent↔Server 之间的 wire types（`Metrics`、`HardwareSnapshot`、`FlowRecord`、`NetFlowReport` 等），确保两端结构体契约一致。
- `android/app/src/main/java/com/aiops/monitor` 是基于 Jetpack Compose 的移动端 UI，复用同一套 `/api/v1/*` 接口；`plugins/*.py` 通过 `plugin_sdk.py` 暴露的 SDK 被 Agent 的 Python Runner 动态加载。
- 构建产物输出到 `bin/`，由 `build.ps1` 通过 `go build -ldflags "-X main.appVersion=..."` 注入版本，Dockerfile 与 `docker-compose.yml` 提供一键编排。
