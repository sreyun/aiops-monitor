## 1. 构建系统总览

本项目采用**多语言、多产物**的混合构建模型：
- Go（Server + Agent）：`go build` + `ldflags` 注入版本号，支持 CGO=0 静态编译
- Docker 多阶段镜像：`docker/Dockerfile` 使用 BuildKit + buildx 实现 linux/amd64 + linux/arm64 双架构
- Android（Kotlin/Compose）：Gradle 8.5.2 + Kotlin 1.9.24
- Python 插件：`pip install -r plugins/requirements.txt`
- CI（GitHub Actions）：推送 `v*` 标签自动触发多架构镜像构建并推送到华为云 SWR

版本来源统一从 Git tag 解析，通过 `-X main.appVersion=<tag>` 注入二进制。

## 2. 核心构建入口

| 入口 | 用途 | 关键参数 |
|---|---|---|
| `build.ps1` | Windows 本地开发一键构建 server+agent，可选 `-CrossCompile` 生成 Linux/macOS 二进制 | `$env:GOOS/GOARCH` 切换目标平台 |
| `docker/Dockerfile` | 多阶段镜像：builder → server / agent；同时产出所有平台的 agent 分发包到 `/out/dist/` | `TARGETOS/TARGETARCH`、`VERSION`、`BASE_REGISTRY` |
| `docker-compose.yml` | 本地编排 aiops-server + victoriametrics + postgres + 可选 aiops-agent | 环境变量 `AIOPS_*` 驱动配置 |
| `.github/workflows/release.yml` | 触发条件 `push.tags: v*`，QEMU + buildx 构建双架构镜像，HMAC-SHA256 计算 SWR 登录密码 | `PLATFORMS=linux/amd64,linux/arm64` |

## 3. 依赖管理策略

- **Go 模块**：`go.mod` 仅声明 3 个第三方库（lib/pq、ledongthuc/pdf、skip2/go-qrcode），并使用 `vendor/` 目录锁定依赖，Docker 构建时以 `-mod=vendor` 离线编译，确保可重复性。
- **Android**：顶层 `android/build.gradle` 声明应用与 Kotlin 插件版本，子项目 `app/build.gradle` 管理具体依赖。
- **Python 插件**：`plugins/requirements.txt` 仅要求 `psutil>=5.9`，Agent 容器内通过 `pip3 install --no-cache-dir --break-system-packages psutil` 安装。

## 4. 多架构与交叉编译

- Dockerfile 在 builder 阶段对当前 `BUILDPLATFORM` 执行 native cross-compile，无需 QEMU 即可产出 arm64 二进制。
- 额外用纯 `GOOS/GOARCH` 循环构建出所有平台的 agent 分发文件（linux-amd64/arm64、darwin-amd64/arm64、windows-amd64），随 server 镜像一起打包，供服务端 `/dl/` 接口下发。
- CI 通过 `setup-qemu-action` + `setup-buildx-action` 在 GitHub 上完成同样的多架构构建。

## 5. 制品与发布流程

```
git push --tags vX.Y.Z
  → GitHub Actions (release.yml)
    → docker buildx multi-platform
    → HMAC-SHA256 生成 SWR 登录凭据
    → 推送 aiops-server:latest + aiops-agent:latest 至 swr.cn-east-3.myhuaweicloud.com/sreyun
```

- 本地也可用 `deploy.sh` 将已编译的 `aiops-server.exe` 直接 scp 到远端主机并通过 systemctl/nohup 重启服务（面向 Windows 宿主的快速部署脚本）。
- 预构建的二进制集中在 `bin/` 目录，命名约定为 `<组件>-<平台>[-架构]`（如 `aiops-server-linux-arm64`）。

## 6. 开发者规范

1. **版本号注入**：新增或修改 `main.appVersion` 注入点时，保持与 `build.ps1` 和 `Dockerfile` 中 `-X main.appVersion=${VERSION}` 一致。
2. **依赖更新**：Go 依赖应同步更新 `go.mod` 并重新 `go mod vendor`，保证 Docker 构建不联网。
3. **跨平台构建**：新增平台需同时更新 `Dockerfile` 中的分发构建循环与 `build.ps1` 的 CrossCompile 分支。
4. **Android 构建**：需在宿主安装 Android SDK/NDK 后通过 `./gradlew assembleRelease` 构建 APK，不在仓库中提交中间产物。
5. **插件依赖**：新增 Python 插件依赖请同步写入 `plugins/requirements.txt`，并在 Agent 镜像构建步骤中安装。
