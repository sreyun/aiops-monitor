## 构建系统概览

本项目采用**多语言、多产物**的构建体系：Go（Server + Agent）、Kotlin/Compose（Android Dashboard）、Python（插件），通过 Docker 多阶段构建统一打包，CI 自动推送至华为云 SWR。

### 1. Go 构建链

- **版本注入**：所有二进制均通过 `-ldflags="-X main.appVersion=${VERSION}"` 注入 Git tag 版本号，本地开发使用 `dev-<commit>` 或 `dev`。
- **交叉编译**：
  - 本地 PowerShell 脚本 `build.ps1` 支持一键构建 Windows 二进制，并可选交叉编译 Linux/macOS 产物到 `bin/`。
  - Docker 生产镜像在 `builder` 阶段基于 `$BUILDPLATFORM` 原生编译，再通过 `GOOS/GOARCH` 为 Server 和 Agent 分别产出目标架构二进制；同时额外构建全平台 Agent 安装器（linux-amd64/arm64、darwin-amd64/arm64、windows-amd64）放入 `/out/dist/`，由 Server 的 `/dl/` 端点分发。
- **依赖管理**：启用 `vendor/` 目录，构建时使用 `GOFLAGS=-mod=vendor`，确保离线可复现。
- **CGO 关闭**：`CGO_ENABLED=0` 保证静态链接、零运行时依赖。

### 2. Docker 多阶段镜像

- **`docker/Dockerfile`**：生产镜像，双 target（`server` / `agent`），默认使用华为云 SWR 镜像加速，支持 `BASE_REGISTRY=docker.io` 切换回官方源。
- **`docker/Dockerfile.dev`**：开发镜像，跳过 dist 全平台交叉编译以加速本地迭代。
- **镜像策略**：
  - Server 镜像仅含单二进制 + 内嵌前端 + 压缩后的 `plugins.zip`，体积最小化。
  - Agent 镜像额外安装 Python3 + psutil 供插件运行。
  - 健康检查通过 `/healthz` 探针。

### 3. Docker Compose 编排

- 一键启动完整栈：`aiops-server` + `victoria-metrics`（时序指标）+ `postgres/pgvector`（关系数据）。
- 环境变量驱动配置：`AIOPS_VM_URL`、`AIOPS_POSTGRES_DSN`、`AIOPS_SECRET_KEY`、`AIOPS_TLS_CERT/KEY`、`AIOPS_FORWARD_LISTEN` 等。
- 数据持久化：`./data`（录制/TLS）、`./vm-data`、具名卷 `aiops-pgdata`。

### 4. CI/CD 流水线（GitHub Actions）

- **触发条件**：推送 `v*` 标签或手动 `workflow_dispatch`。
- **多架构构建**：`linux/amd64, linux/arm64`，使用 QEMU + buildx。
- **SWR 认证**：通过 AK/SK HMAC-SHA256 动态生成登录凭据，无需预存密码。
- **缓存**：GHA cache 加速 Go module 下载。
- **产物**：同时打 `:version` 和可选 `:latest` 标签推送到 `swr.cn-east-3.myhuawei.com/sreyun/aiops-{server,agent}`。

### 5. Android 构建

- Gradle 8.5.2 + Kotlin 1.9.24，模块位于 `android/app/build.gradle`，独立于 Go 构建流程。

### 6. 部署脚本

- `deploy.sh`：Windows 服务端的一键 SSH 部署脚本（上传 → 停止旧进程 → 替换 → 重启），适用于无容器的裸机场景。

## 关键约定

| 约定 | 说明 |
|------|------|
| 版本号来源 | Git tag（`v*`），未打 tag 时退化为 `dev-<short-commit>` |
| 依赖锁定 | `go mod vendor` + `-mod=vendor`，禁止网络拉取 |
| 二进制命名 | `aiops-server` / `aiops-agent`，带平台后缀用于分发包 |
| 端口固定 | Server 默认 8529，VM 8428，Agent 转发 10100-10300 |
| 配置落库 | 所有配置、用户、审计、事件、工单、会话均持久化到 PG，不再使用内置 SQLite |

## 开发者注意事项

- 修改 Go 代码后，本地快速验证用 `docker compose up --build`（走 dev 镜像）；跨平台发布前务必跑 `build.ps1 -CrossCompile` 或触发 release workflow。
- 新增第三方 Go 依赖需同步更新 `vendor/`（`go mod vendor`），否则 CI 会失败。
- 插件目录 `plugins/` 会在构建时被 zip 进 server 镜像，新增插件需重新构建镜像。
- Android 侧改动需在 `android/` 目录下单独用 Android Studio 或 `gradlew assembleDebug` 构建。
