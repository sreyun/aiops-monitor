## 1. 系统概览

本项目采用**多语言、多产物**的构建体系：Go（Server/Agent）、Kotlin Jetpack Compose（Android）、Python（插件），通过 Docker + GitHub Actions 实现跨平台编译、镜像打包与制品分发。版本注入统一通过 Go ldflags `-X main.appVersion` 完成，所有二进制均内嵌版本号。

## 2. 核心构建工具与入口

- **本地开发构建**
  - Windows: `build.ps1` — 读取 Git tag 作为版本号，输出 `bin/aiops-server.exe`、`bin/aiops-agent.exe`；支持 `-CrossCompile` 参数交叉编译 Linux/macOS 二进制。
  - Linux/macOS: `deploy.sh` 仅负责部署，无独立构建脚本；开发者可直接 `go build ./cmd/server` / `./cmd/agent`。
- **容器化构建**
  - `docker/Dockerfile` — 多阶段、多目标（server/agent）Dockerfile，使用 `--platform=$BUILDPLATFORM` 在构建机上原生交叉编译，避免 QEMU 模拟 Go 编译；同时为一键安装包预编译全部 Agent 平台二进制（linux/amd64+arm64、darwin/amd64+arm64、windows/amd64）。
  - `docker/Dockerfile.dev` — 开发用镜像，配合 `docker-compose.yml` 使用。
- **CI/CD**
  - `.github/workflows/release.yml` — 推送 `v*` 标签或手动触发时，使用 `docker/setup-buildx-action` + `QEMU` 构建 linux/amd64 + linux/arm64 双架构镜像，推送到华为云 SWR（`swr.cn-east-3.myhuaweicloud.com/sreyun`），并生成 step summary。

## 3. 依赖管理策略

| 组件 | 方式 | 说明 |
|------|------|------|
| Go (server/agent) | `go.mod` + `vendor/` | 使用 `GOFLAGS=-mod=vendor` 锁定依赖，Dockerfile 中 COPY vendor 目录参与构建，确保离线可复现 |
| Android | Gradle (`android/build.gradle`) | Kotlin 1.9.24 + AGP 8.5.2，仓库源 google()/mavenCentral() |
| Python 插件 | `plugins/requirements.txt` | 可选依赖 `psutil>=5.9`，Linux 下由 Go 核心采集指标可不装 |

## 4. 版本与制品命名约定

- **版本号来源**: Git tag（如 `v5.5.0`），未打 tag 则回退到 `dev-<short-commit>`。
- **ldflags 注入**: `-ldflags="-s -w -X main.appVersion=${VERSION}"`，所有二进制运行时可查询自身版本。
- **二进制命名**:
  - 服务端: `aiops-server` / `aiops-server.exe`
  - Agent: `aiops-agent` / `aiops-agent.exe`
  - 交叉产物: `<name>-<os>-<arch>`（如 `aiops-agent-linux-amd64`、`aiops-agent-darwin-arm64`）
- **Docker 镜像**:
  - `swr.cn-east-3.myhuaweicloud.com/sreyun/aiops-server:<tag>`
  - `swr.cn-east-3.myhuaweicloud.com/sreyun/aiops-agent:<tag>`
  - 每次 release 同时打上 `latest` 标签（可配置）。

## 5. 多架构与跨平台编译

- **Go 交叉编译**: 通过设置 `GOOS`/`GOARCH` 环境变量，在单一主机上编译出任意平台二进制（`build.ps1` 显式设置 GOOS=darwin/linux，Dockerfile 中由 buildx 自动传入 TARGETOS/TARGETARCH）。
- **Docker Buildx**: CI 中使用 `docker/setup-buildx-action` 配合 `QEMU` 实现 linux/amd64 + linux/arm64 双架构镜像构建，无需在 ARM 机器上执行 Go 编译。
- **CGO 禁用**: 所有 Go 构建均带 `CGO_ENABLED=0`，确保静态链接、零运行时依赖。

## 6. 制品分发与部署

- **预构建镜像**: 华为云 SWR 提供已构建好的 server/agent 镜像，用户可直接 `docker compose up -d` 拉取运行。
- **本地一键部署**: `deploy.sh` 通过 scp + ssh 将本地编译的二进制上传至远端服务器，替换后重启服务（兼容 systemd 与非 systemd 环境）。
- **Docker Compose 编排**: `docker-compose.yml` 定义 aiops-server、victoriametrics、postgres、aiops-agent 四服务，默认端口 8529（Web/API）、8428（VM UI，注释掉）、10100-10300（TCP 转发范围）。

## 7. 开发者应遵循的规则

1. **新增 Go 依赖** → 更新 `go.mod` 并执行 `go mod vendor`，确保 `vendor/` 同步提交。
2. **修改版本号** → 打 Git tag（`git tag vX.Y.Z && git push --tags`），CI 会自动构建并发布镜像。
3. **新增平台支持** → 在 `docker/Dockerfile` 的多平台构建块中添加对应 `GOOS/GOARCH` 组合，并在 `build.ps1` 的 CrossCompile 分支补充。
4. **Android 客户端** → 通过 Android Studio 打开 `android/` 目录构建，Gradle 版本固定于根级 `build.gradle`，不要随意升级。
5. **插件扩展** → 新增 Python 依赖需同步更新 `plugins/requirements.txt`，并确保 Agent 容器启动时 `pip3 install` 成功。
6. **敏感配置** → 生产环境必须修改 `AIOPS_SECRET_KEY` 和 PostgreSQL 密码，二者需在 `docker-compose.yml` 中保持一致。
