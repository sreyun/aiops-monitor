本仓库为 Go + Android + Python 插件的混合工程，各子模块采用各自生态的标准依赖管理方式，整体呈现“最小化、无 vendor、无私有源”的特点。

- **Go（Server/Agent）**
  - 使用 go.mod + go.sum 进行依赖声明与版本锁定，Go 版本固定为 1.22。
  - 当前仅声明了 3 个第三方包：github.com/lib/pq（PostgreSQL 驱动）、github.com/skip2/go-qrcode（二维码生成）、github.com/ledongthuc/pdf（PDF 生成），其余功能均为标准库或内嵌实现。
  - 未启用 vendor/ 目录（目录为空），也未配置 GOPROXY/GOPRIVATE/GONOSUMDB 等代理或私有源环境变量，构建时直接拉取官方 Go Module Proxy。
  - go.sum 中所有条目均已校验哈希，具备可重复构建基础。

- **Android（Kotlin + Compose）**
  - 通过 android/app/build.gradle 中的 dependencies {} 块声明依赖，使用 Maven Central / Google Maven。
  - 关键依赖包括：Compose BOM 2024.09.00、Retrofit 2.11.0 + OkHttp 4.12.0、DataStore Preferences 1.1.1、Navigation Compose 2.8.0 等。
  - 顶层 gradle.properties 仅设置 JVM 参数与 AndroidX/Kotlin 开关，未配置 Gradle 镜像或私有仓库。
  - android/gradle/wrapper/gradle-wrapper.properties 由 Gradle Wrapper 管理，无需额外锁文件。

- **Python 插件**
  - plugins/requirements.txt 仅声明一个可选依赖 psutil>=5.9，并附带注释说明 Linux 平台可由 Go 核心原生采集指标，可不安装 psutil。
  - 插件 SDK 与示例脚本均位于本地 plugins/ 目录，无独立 setup.py/pyproject.toml，以脚本形式被 Server 动态加载。

- **约定与建议**
  - Go 侧保持极小依赖面，新增依赖需同步更新 go.mod 并通过 go mod tidy 维护 go.sum。
  - Android 侧建议统一通过 Compose BOM 管理版本，避免手动指定冲突版本。
  - 若未来引入私有 Go 模块或国内镜像，应在 CI 环境注入 GOPROXY/GOPRIVATE 环境变量，而非修改源码。
