# NetFlow 被动接收优先于主动采集

_来源：a8b268a → 791b7da 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
网络设备（防火墙/交换机）推送 NetFlow 到 Agent 比 Agent 轮询设备 API 更高效且对设备压力更小，但需要设备侧预先配置流出口。

## 决策驱动
- 设备负载
- 实时性
- 部署复杂度

## 备选方案
- **仅主动采集（SNMP/REST 轮询）** _（已否决）_ — 优点：无需改设备配置；实现相对简单；缺点：轮询间隔长导致粒度粗；大量设备时轮询风暴
- **被动接收 + 主动采集双模式** — 优点：被动模式零侵入、低延迟；主动模式作为兜底；缺点：实现复杂度高；需维护两套解析逻辑

## 决策
P0 先实现 UDP 被动接收（v5/v9 解析 + 内存聚合窗口），P1 再扩展 SNMP/REST 主动采集作为补充。两者共享同一 `FlowRecord` 结构和 `/api/v1/agent/netflow` 端点，通过 source 字段区分来源。

## 影响
被动模式需要设备侧配合配置流出口到 Agent IP:2055；主动模式可作为无法改设备配置场景的降级方案。聚合器有 100K flow 内存上限和背压丢弃计数保护。
