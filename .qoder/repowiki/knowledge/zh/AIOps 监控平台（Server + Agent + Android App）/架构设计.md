仓库采用「双二进制 + 多语言插件」架构：
- `cmd/server` 为单体 HTTP 服务入口（`main.go`），通过 `handlers.go` 中的 `http.ServeMux` 按 Go 1.22 方法路径模式注册 `/api/v1/*` REST 路由，内嵌 `web/` 静态资源并统一挂载 CORS、安全头、gzip、body 限流等中间件链；启动时强制要求 PostgreSQL（关系数据）和 VictoriaMetrics（时序数据）两个外部存储，拒绝回落到内置 aiops.db。
- `cmd/agent` 为部署在目标主机的采集器，支持 relay 网关模式、多服务端广播上报、Python 插件执行器，以及 Redfish/NetFlow/PacketCapture 三类可选采集器（通过配置字段 `redfish_targets`、`netflow`、`packet_capture` 按需启用）。
- `shared/` 提供跨二进制的共享类型（如 `Metrics`）。
- `plugins/` 为 Python 生态（`plugin_sdk.py` + `core_metrics.py` + 业务插件），由 agent 通过子进程调用。
- `android/app` 为独立 Kotlin/Compose 客户端，使用 Retrofit 对接同一套 `/api/v1/*` 接口。
- 运行时后台协程：`Notifier`（告警评估去重）、`checkRunner`（HTTP/TCP 拨测）、`apiRunner`（API 性能监控）、`playbookManager`（定时编排）、`sloManager`（SLO 燃烧）、`vmWriter`（VM remote-write pump）、`termManager`（终端录制回放）、`hermes`（AI 诊断）等，均通过 `NewServer` 组装并由 `main` 启动 goroutine 驱动。
- 构建产物集中在 `bin/`，由根目录 `build.ps1` 通过 `-ldflags -X main.appVersion=...` 注入 Git tag 版本号，同时产出 server/agent 的 linux/darwin/arm64/amd64 交叉编译包。
