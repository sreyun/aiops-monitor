### 拨测
- 定义：主动式健康检查，系统定时向目标 URL/端口/主机发起探测请求，检测可用性、响应时间、状态码等指标，异常时触发告警。支持 HTTP/TCP/Ping/进程存活四种类型，结果写入 VictoriaMetrics 并支持历史曲线回看。
- 别名：主动监控、探活

### 剧本
- 定义：自动化运维编排工具，将多条 shell 命令按顺序组织成可执行的步骤序列，支持批量并行执行到多台主机，具备超时控制、失败处理、执行历史审计等功能。命令经 Agent 反向通道下发，非交互式执行。
- 别名：Playbook、自动化脚本

### 中继模式
- 定义：网关代理部署模式，在内网仅一台机器可联网的场景下，该机器安装 Relay 模式的 Agent，将所有内网 Agent 的请求反向代理到云端监控中心，实现网络穿透。与多服务端推送互斥。
- 别名：Relay、网关代理

### 机器指纹
- 定义：基于 machine-id + 主 MAC 地址的 SHA-256 前 12 位生成的唯一标识，用于 Agent 注册时的身份绑定。后续所有上报与终端通道请求均携带指纹鉴权，Token 轮换后已装 Agent 无需更新配置，防止克隆镜像导致的身份冲突。
- 别名：host_id、设备指纹

### SRE中枢
- 定义：统一的事件驱动工作流平台，整合告警→事件→自动修复→工单的完整闭环流程。包括事件管理（告警/SLO/手动汇聚+时间线）、自动修复（护栏+审批）、SLO/错误预算、工单系统等核心能力。
- 别名：事件中心、故障管理

### 向量化模型
- 定义：专门用于 RAG（检索增强生成）的嵌入模型，与对话模型解耦配置。支持任意 OpenAI 兼容的 `/embeddings` 接口，可将文本转换为向量存入 PostgreSQL pgvector，用于历史相似案例检索。维度必须与建表时的 vector(N) 列一致。
- 别名：embedding模型、RAG模型

### 告警治理
- 定义：通知下发前的决策层，对 firing 告警执行静默（时段/星期临时屏蔽）、抑制（主因抑制衍生告警）、路由（按级别·主机分流渠道）三大策略，从源头抑制告警风暴、降低夜间打扰、按业务分流。恢复通知一律照发不受静默影响。
- 别名：告警过滤、通知策略

### 统一消息中心
- 定义：聚合 SRE 工作流与 AI 产生的各类通知的统一收件箱，包含事件/告警/SLO突破/自动修复/AI巡检/工单等消息类型，支持未读计数、深链跳转、一键已读/全部已读，持久化于 PostgreSQL kv_state。
- 别名：消息聚合、通知中心

### SSRF出站防护
- 定义：针对 AI 端点与通知 Webhook 等用户可影响 URL 的出站请求的安全守卫机制，默认拒绝云元数据（169.254.169.254 等）与链路本地地址；设置 AIOPS_SSRF_STRICT=true 时额外拒绝环回与 RFC1918 私网地址，覆盖重定向与 DNS rebinding 攻击。
- 别名：出站守卫、safedial
