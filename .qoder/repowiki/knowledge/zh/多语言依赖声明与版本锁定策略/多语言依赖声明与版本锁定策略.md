本仓库为 Go + Android(Kotlin) + Python 插件的多语言项目，各子模块采用各自生态的依赖管理方式，未使用统一的跨语言依赖工具。

Go 后端（Server/Agent）
- 使用 go.mod + go.sum 进行依赖声明与版本锁定，Go 版本固定为 1.22。
- 当前仅声明了 3 个第三方包：github.com/lib/pq（PostgreSQL 驱动）、github.com/skip2/go-qrcode、github.com/ledongthuc/pdf，其余均为标准库。
- 存在 vendor/ 目录但为空，未启用 vendoring；也未发现 GOPRIVATE 或私有代理配置。
- 所有依赖均通过公共 Go 模块代理拉取，无私有仓库或镜像源设置。

Android 移动端
- 使用 Gradle + Kotlin DSL，顶层 android/build.gradle 声明 Google/MavenCentral 仓库及插件版本。
- 应用级 android/app/build.gradle 集中声明 Compose BOM、Retrofit、OkHttp、DataStore 等依赖，版本号硬编码在 build 脚本中。
- 未发现 gradle.properties 中的私有仓库或代理配置。

Python 插件层
- plugins/requirements.txt 仅声明可选依赖 psutil>=5.9，用于非 Linux 平台的基础指标采集。
- 核心指标由 Go 原生采集，Python 侧仅为可选扩展，无严格版本锁定。

约定与建议
- 新增 Go 依赖后需同步更新 go.sum，避免 vendor 模式导致构建不一致。
- Android 依赖建议统一通过 Compose BOM 管理，避免版本冲突。
- 若引入私有 Go 模块或 Maven 仓库，应在对应配置文件（go.mod、gradle.properties）中显式声明，而非依赖全局环境变量。
