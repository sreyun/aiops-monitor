仓库为单体多包 Go 工程（module `aiops-monitor`），按运行角色拆分为两个独立可执行入口：
- `cmd/server/main.go`：HTTP/WS API 服务，通过 `Server.Routes()` 用 Go 1.22 方法路径模式注册 `/api/v1/*` 路由；中间件链为 `securityHeaders → cors → gzip → bodyLimit → auth`；内置静态资源通过 `//go:embed web` 注入，前端单页由 `/app.js` 动态拼接多个 JS 模块。
- `cmd/agent/main.go`：采集端，支持 relay 网关模式与多服务器广播上报；内置跨平台 Collector 接口（`collector_linux/darwin/windows/other.go` 构建标签选择实现）+ Python 插件 Runner；新增 Redfish/NetFlow/PacketCapture 三类采集器配置字段，由 Agent 启动时按需拉起。
- 存储层：`store.go` 维护内存 Host ring buffer + 三级历史降采样（raw/1m/5m），并通过 `pgstore.go` 将审计日志、事件、告警、用户、会话等关系数据持久化到 PostgreSQL（JSONB 表 + pgvector 向量索引）；时序数据统一写入 VictoriaMetrics（`vmWriter` + `AIOPS_VM_URL`）。
- `shared/wire.go` 定义跨进程共享的 `Report/Sample/Metrics` 数据结构，是 Agent↔Server 协议契约。
- `android/app/src/main/java/com/aiops/monitor/` 使用 Jetpack Compose + MVVM（`ui/screens` + `ui/viewmodel` + `data/store` + `data/models`），通过 `ApiClient.kt` 调用同一套 `/api/v1/*` REST 接口。
- 依赖方向：Android ← HTTP → Server ← PG/VM；Agent → HTTP → Server；Server 不反向依赖 Agent。
