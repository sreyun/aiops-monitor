- 所有对外 API 统一以 `POST/GET /api/v1/...` 命名，并在 `Server.Routes()` 中集中注册，避免分散的 `http.HandleFunc` 调用。
- 跨进程共享的数据结构集中在 `shared/` 包（如 `Report`、`Sample`、`Metrics`），Agent 与 Server 各自 import 同一份类型定义，保证协议一致性。
- 平台差异通过 Go 构建标签文件组织（`*_linux.go`、`*_windows.go`、`*_darwin.go`、`*_other.go`），在同一个 package 内按 OS 编译出不同实现。
- 对敏感请求体一律包裹 `MaxBytesReader`（`maxBodyBytes = 100MiB`），并通过 `securityHeadersMiddleware` 统一设置 X-Content-Type-Options、CSP、Frame-Options 等安全头。
- PostgreSQL 连接失败时采用带退避的重试循环（`mustOpenPG` 最多 10 次 × 2s），冷启动 docker-compose 场景下允许 PG 延迟就绪。
- 响应压缩通过 `gzipResponseWriter` + `sync.Pool` 复用 `gzip.Writer`，并对 WebSocket 升级、terminal/forward/proxy 流式路径显式跳过压缩。
