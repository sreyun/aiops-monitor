仓库采用「单 Go 模块 + 多入口」结构：
- `cmd/server/main.go` 启动 HTTP 服务，通过 `handlers.go` 中的 `Server.Routes()` 以 Go 1.22 方法+路径模式注册 `/api/v1/*` REST 路由；静态前端通过 `//go:embed web` 内嵌，由单一 `/app.js` 拼接多个 JS 模块按依赖顺序输出。
- 存储双后端强制化：PostgreSQL（关系型数据，`mustOpenPG` 带重试连接）+ VictoriaMetrics（时序指标，`AIOPS_VM_URL` 必填），内置 aiops.db 已停用。
- `cmd/agent/main.go` 为跨平台采集器，支持 relay 网关模式、日志加密上报、安全环境自检（麒麟 kysec / SELinux / AppArmor / firewalld / Defender / SIP）；基础指标通过 `Collector` 接口 + build tag（`collector_linux.go`/`collector_windows.go`/`collector_darwin.go`/`collector_other.go`）实现平台差异化。
- 三类新增采集器作为独立文件注入 Agent 生命周期：`collector_redfish.go`（BMC/iDRAC/iLO Redfish 轮询，兼容旧固件 TLS 1.0/RSA cipher）、`collector_netflow.go`（UDP NetFlow v5/v9 接收 + 五元组聚合窗口）、`collector_packet.go`（Linux nf_conntrack diff 增量流）。三者均通过 `reporter` 回调向 Server 推送 `HardwareReport` / `NetFlowReport`。
- `shared/wire.go` 定义 agent↔server 共享 wire types（`Metrics`、`HardwareSnapshot`、`FlowRecord`、`NetFlowReport` 等），保证协议契约不漂移。
- `plugins/` 目录为 Python 插件生态（`plugin_sdk.py` + `core_metrics.py` + 示例插件），Agent 通过子进程调用执行。
- `android/app/` 为 Kotlin/Jetpack Compose 客户端，复用同一套 API。
- 构建产物在 `bin/`（预编译二进制 + plugins.zip），Docker 镜像由 `docker/Dockerfile` 生成，CI 通过 `.github/workflows/{codeql,release,security}.yml`。
