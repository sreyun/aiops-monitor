- 平台差异通过 Go build tag 拆分同名文件（如 `collector_linux.go`/`collector_windows.go`/`collector_darwin.go`/`collector_other.go`），由编译器选择实现。
- 所有对外 wire type 集中在 `shared/wire.go`，agent 与 server 共用同一份 struct 定义，避免 JSON 契约漂移。
- HTTP 中间件链统一包裹在 `main()` 中：`securityHeadersMiddleware → corsMiddleware → gzipMiddleware → bodyLimitMiddleware → authMiddleware`，WS/proxy/streaming 路径显式跳过压缩。
- 外部依赖初始化采用「带重试的连接」模式（`mustOpenPG` 最多 10 次、间隔 2s），冷启动失败后 fatal，不允许静默降级到本地文件。
- 长周期后台任务以 goroutine + ticker 形式在 `main()` 中启动（notifier、checks、apimon、scheduler、SLO evaluator、VM writer、AI inspection），并通过 `signal.Notify` 优雅关闭。
- Redfish/OceanStor/NetFlow/Packet 等扩展采集器均以独立文件 + 配置字段（`redfish_targets`、`oceanstor_targets`、`netflow`、`packet_capture`）按需启停，未配置则不注册。
