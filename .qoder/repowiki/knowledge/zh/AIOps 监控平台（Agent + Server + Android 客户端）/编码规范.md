- 跨进程协议类型（Report/Sample/Event/Metrics 等）统一定义在 `shared/` 包，Agent 与 Server 各自 import 同一份结构体，避免两端结构漂移。
- HTTP 路由集中注册于 `Server.Routes()`，使用 Go 1.22 的 `mux.HandleFunc("POST /api/v1/...", s.handleXxx)` 语法，按功能域（agent/alerts/users/playbooks/sre/hermes/forward 等）分组排列。
- 中间件以闭包形式组合：`securityHeadersMiddleware(corsMiddleware(gzipMiddleware(bodyLimitMiddleware(authMiddleware(...))))`，每个中间件只关心单一横切关注点（CORS/压缩/限流/鉴权）。
- 平台差异通过 Go build tag 文件拆分：`*_linux.go`/`*_windows.go`/`*_darwin.go`/`*_other.go` 同名文件共存，编译期选择具体实现（如 collector、pty、console）。
- PostgreSQL 持久化采用 JSONB 列 + 自描述 Go struct 的方式（`data JSONB NOT NULL`），迁移脚本集中在 `pgstore.go` 的 `migrate()` 中，无需手写 DDL 迁移工具。
- 告警/事件去噪统一通过 `lastEvent[key]` 时间窗口抑制（`eventCooldownSec=300s`），key 由 hostID|source|level|message 组成，超出容量时按过期时间清理。
