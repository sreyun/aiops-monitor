仓库采用「多入口单模块」结构：`cmd/server/main.go` 与 `cmd/agent/main.go` 是两个独立可执行程序的入口，共享 `shared/` 包定义 `Report`/`Sample`/`Event` 等跨进程协议。

- Server 层（`cmd/server/`）：以 `Server` 聚合体为核心（见 `handlers.go`），通过 `Routes()` 用 Go 1.22 方法+路径模式注册所有 `/api/v1/*` REST 端点；中间件链为 `securityHeaders → cors → gzip → bodyLimit → auth`。存储双后端：内存 `Store`（ring buffer + 三档降采样 histRaw/hist1m/hist5m，见 `store.go`）+ PostgreSQL（`pgstore.go`，JSONB 持久化 incidents/tickets/audit/events/hosts/kv_state/terminal_recordings/diagnosis_embeddings/ai_memory_embeddings/hermes_rules/templates/sessions 等表）。可选 VictoriaMetrics remote-write（`vmWriter`）承载时序指标。
- Agent 层（`cmd/agent/`）：`Collector` 接口由 `collector_linux.go`/`collector_windows.go`/`collector_darwin.go` 按 build tag 实现，负责 CPU/Mem/Disk/GPU/Net 基础指标；Python 插件通过 `plugin_sdk.py` 扩展自定义采集；内置 Playbook 模块（`modules.go`）提供 `gather_facts/service/package/copy` 四步抽象，服务端通过 `__AIOPS_MODULE__` 封套 JSON 下发。
- 三类新增采集器（按 coding_intent 规划）：Redfish（`collector_redfish.go`，轮询 BMC/iDRAC/iLO 硬件健康）、NetFlow（`collector_netflow.go`，UDP v5/v9 接收并聚合五元组 Flow）、PacketCapture（通过 `/proc/net/nf_conntrack` 差值计算增量连接，见 `config.example.json` 中 `packet_capture` 字段）——三者均通过 `reporter.go` 中的 `hardwareReporter`/`netflowReporter`/`packetReporter` 上报到 Server 对应 `/api/v1/agent/hardware`、`/api/v1/agent/netflow` 端点。
- Android 客户端（`android/app/src/main/java/com/aiops/monitor/`）：Kotlin + Jetpack Compose，按 data/ui/components/screens/viewmodel/theme 分层，通过 `ApiClient`/`ApiService` 调用同一套 `/api/v1/` REST，Terminal 使用 WebSocket 复用 Server 的终端通道。
- 前端静态资源通过 `//go:embed web` 内嵌进二进制，`/app.js` 在运行时将 `web/js/*.js` 多个源模块拼接成单个脚本以利用函数提升顺序。
