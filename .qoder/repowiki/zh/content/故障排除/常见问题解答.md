## 简介
本 FAQ 面向 AIOps Monitor 的用户与运维人员，聚焦安装部署、配置错误、网络连接、权限安全、端口转发与 HTTP 代理、日志采集、告警通知、终端会话、性能稳定性等高频问题。每个问题提供症状描述、原因分析与解决步骤，并附带错误信息解释与快速定位方法，按类型组织便于快速查找。

## 问题分类与快速导航
- 安装部署类：Docker 启动失败、镜像拉取受限、二进制运行报错、开机自启异常
- 配置错误类：环境变量缺失、配置文件字段错误、密钥未设置、存储后端未就绪
- 网络连接与代理类：Agent 无法连接服务端、跨网络访问失败、反向代理超时
- 权限与安全类：/proc 读取受限、SELinux/AppArmor/kysec 拦截、TLS 证书校验失败
- 端口转发与 HTTP 代理类：TCP/UDP 映射不可用、HTTP 代理“unexpected EOF”、WebSocket 升级失败
- 日志采集与检索类：日志路径无效、加密上报失败、全文检索无结果
- 告警与通知类：飞书/钉钉/邮件/短信/语音电话推送失败、阈值不生效
- 终端与会话类：远程终端打不开、会话录制回放异常、只读旁观不可用
- 性能与稳定性类：高并发下内存增长、响应慢、会话泄漏

## 安装部署类问题
- 症状
  - Docker Compose 启动后服务立即退出或反复重启
  - 浏览器访问 http://localhost:8529 提示连接拒绝
  - 直接运行二进制时报错退出
- 可能原因
  - 缺少必填的环境变量（PostgreSQL DSN、VictoriaMetrics URL）
  - 依赖容器（postgres/victoriametrics）未就绪或镜像拉取失败
  - 端口被占用或未正确映射
  - 二进制 dist 目录缺失导致一键安装脚本找不到 Agent 包
- 解决步骤
  - 检查环境变量是否已配置 AIOPS_POSTGRES_DSN 与 AIOPS_VM_URL；若未配置，服务会拒绝启动并输出明确错误提示
  - 查看 docker-compose 日志，确认 postgres 与 victoriametrics 健康状态；必要时配置镜像加速器或替换为 SWR 镜像（仅 AMD64）
  - 确认本地端口 8529 未被占用；如需修改，在 compose 或服务端参数中调整监听地址
  - 若使用二进制运行，确保程序所在目录或 -dist 指定目录下存在对应架构的 Agent 二进制，否则一键安装功能不可用

## 配置错误类问题
- 症状
  - 启动时打印“AIOPS_POSTGRES_DSN 未配置”或“AIOPS_VM_URL 未配置”
  - 首次登录强制安全初始化后仍无法进入面板
  - 告警推送测试失败
- 可能原因
  - 未设置必要的环境变量或配置文件字段缺失
  - 未设置 AIOPS_SECRET_KEY，导致敏感配置以明文落库（建议启用静态加密）
  - 账户初始凭据未修改或 MFA 策略未配置
- 解决步骤
  - 补齐 AIOPS_POSTGRES_DSN 与 AIOPS_VM_URL；服务启动时会重试 PG 连接并在失败时终止
  - 设置 AIOPS_SECRET_KEY 以启用 AES-256-GCM 静态加密，避免明文落库风险
  - 首次登录后完成安全初始化，修改用户名与密码，并按需启用 MFA
  - 告警通道配置完成后点击“发送测试”，逐项验证飞书/钉钉/邮件/短信/语音电话

## 网络连接与代理类问题
- 症状
  - Agent 长时间不上报主机指标
  - 通过 /proxy/{hostID}/{port}/... 访问目标 Web 服务偶尔成功，偶发“无法解析上游响应”
- 可能原因
  - Agent 未配置正确的服务端地址或 Token
  - 跨网络环境需要中继模式（Relay），内网机器直连公网受限
  - HTTP 代理链路中存在竞态条件导致响应截断（已修复）
- 解决步骤
  - 检查 Agent 配置中的 server/servers 字段与命令行 --server/--token；多服务端场景优先使用 servers 数组
  - 内网仅一台机器可联网时，启用 Relay 模式：该机器作为网关中转所有请求到云监控中心
  - 若出现“无法解析上游响应”或“unexpected EOF”，请升级到包含修复的版本，并确保服务端与 Agent 均更新

## 权限与安全类问题
- 症状
  - 部分指标缺失（如 CPU/内存/磁盘 IO/GPU）
  - 启动日志提示“数据采集权限不足”或“/proc 路径无法读取”
  - 安全模块处于 enforcing 模式导致采集失败
- 可能原因
  - 非 root 用户运行 Agent，或被 kysec/SELinux/AppArmor/firewalld/Defender/SIP 等安全模块拦截
  - 未正确配置 CA 证书或跳过 TLS 校验（仅用于实验室/自签）
- 解决步骤
  - 以 root 身份运行 Agent，或在安全模块中添加白名单放行 /proc 相关路径
  - 使用 --security-mode=permissive 自动切换宽容模式（2 小时后恢复 enforcing），或使用 auto/enforcing 模式输出修复命令
  - 若服务端使用自签名证书，可在 Agent 侧配置 --ca-cert 信任 CA 或临时使用 --tls-skip-verify（不建议生产）

## 端口转发与 HTTP 代理问题
- 症状
  - TCP/UDP 映射创建后本地客户端无法连接
  - HTTP 代理访问目标 Web 服务返回“unexpected EOF”或页面不完整
  - WebSocket 升级失败
- 可能原因
  - 监听地址绑定在非回环地址且未做防火墙限制，存在安全风险
  - HTTP 代理链路中 session 生命周期管理不当导致数据流截断（已修复）
  - Nginx 未正确配置 Upgrade 头导致 WebSocket 升级失败
- 解决步骤
  - 将 forward_listen 设置为 0.0.0.0（Docker 部署）并通过防火墙限制来源 IP
  - 升级到包含修复的版本，确保服务端与 Agent 均更新，消除竞态导致的 EOF
  - 在反向代理层配置 WebSocket Upgrade 头，确保 /proxy 路径透传

## 日志采集与检索问题
- 症状
  - 日志采集未生效或检索不到新增日志
  - 加密上报失败或解密失败
- 可能原因
  - 日志路径配置错误或目录不存在
  - 加密上报未正确配置 log_key 或密钥不一致
- 解决步骤
  - 在 Agent 配置中填写 log_paths（文件或目录，逗号分隔），确保路径存在且可读
  - 确认服务端下发的 log_key 长度与算法一致，保持两端一致
  - 在面板「日志检索」中按主机/级别/关键字/时间筛选，观察增量采集效果

## 告警与通知问题
- 症状
  - 告警阈值调整后不生效
  - 飞书/钉钉/邮件/短信/语音电话推送失败
  - 告警风暴频繁
- 可能原因
  - 阈值零值未配置导致回退默认值
  - 通知渠道配置错误或鉴权失败
  - 未启用告警治理（静默/抑制/路由）
- 解决步骤
  - 在「告警设置」中逐项配置阈值，零值会自动回退推荐默认值
  - 逐项测试各通知渠道，确认 Webhook/SMTP/短信/语音电话连通性
  - 启用告警治理：静默规则（时段/星期）、抑制规则（主因抑衍生）、路由分流（严重走电话/钉钉，警告走飞书）

## 终端与会话问题
- 症状
  - 远程终端无法打开或连接中断
  - 会话录制回放卡顿或尺寸错乱
  - 只读旁观无法加入活跃会话
- 可能原因
  - 跨外网未放行 WebSocket 升级头
  - 会话录制持久化未正确写入 PG
  - 权限不足或二次认证未通过
- 解决步骤
  - 在反向代理层放行 WebSocket Upgrade 头，确保 /terminal 路径透传
  - 确认服务端已绑定 PG 并开启会话录制入库，避免重启丢失
  - 检查用户权限与二次认证策略，确保具备终端访问权限

## 性能与稳定性问题
- 症状
  - 多主机轮询时带宽占用高
  - 长连接会话泄漏或内存持续增长
  - 响应延迟升高
- 可能原因
  - 未启用 gzip 压缩
  - 会话空闲超时未清理
  - 最大会话数限制未合理配置
- 解决步骤
  - 启用 gzip 压缩，减少 JSON 响应体积（默认已启用，排除 terminal/forward/proxy 路径）
  - 检查空闲会话清理逻辑，确保 idleChecker 正常运行
  - 根据业务规模调整 maxForwardSessions 与端口范围映射数量

## 故障定位方法与工具
- 快速定位
  - 查看服务端启动日志，确认 PG/VM 连接状态与 TLS 配置
  - 检查 Agent 日志，关注安全模块检测、/proc 访问、转发通道就绪情况
  - 使用 API 健康检查接口验证转发能力与统计信息
- 常用工具
  - curl/wscat/websocat 测试 HTTP/WebSocket 代理
  - netstat/ss 检查本地监听端口与连接状态
  - journalctl/systemd 查看守护进程日志
