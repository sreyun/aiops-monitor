## 简介
本附录为 AIOps Monitor 的辅助资料，聚焦于快速上手、排障与扩展。内容涵盖术语解释、参考链接、许可证说明、常用命令、配置模板、常见问题排查、版本与升级建议、兼容性矩阵以及社区参与方式等，帮助用户高效部署与运维平台。

## 术语表
- Agent：采集端，负责主机指标、日志、进程、GPU 等数据采集并通过反向通道上报服务端。
- 服务端（Server）：单二进制服务，聚合数据、提供面板、告警、终端、转发、SRE 中枢与 AI 巡检诊断能力。
- PostgreSQL（PG）：关系型数据库，用于存储配置、用户、审计、事件、工单、会话等全部关系数据。
- VictoriaMetrics（VM）：时序数据库，用于存储指标与趋势数据。
- 拨测：主动探测，包括 HTTP/TCP/Ping/进程存活等，用于业务可用性监控。
- 剧本（Playbook）：多步骤自动化编排，按目标批量并行执行命令并回传输出。
- 端口转发：将远端主机的 TCP/UDP 端口映射到服务端本地端口，或经 /proxy 进行无状态 HTTP 代理。
- 远程终端：浏览器内 TTY，经 Agent 反向连接，支持录制回放、旁观与审计。
- SRE 中枢：事件管理、自动修复（护栏+审批）、SLO/错误预算、工单闭环。
- AI 巡检诊断：可插拔 LLM + 启发式兜底，结合 RAG 检索相似案例进行根因研判。
- 统一消息中心：事件/告警/SLO/自动修复/AI/工单的统一收件箱。
- 告警治理：静默/抑制/路由，抑制告警风暴并按策略分流通知。
- RBAC/MFA：基于角色的访问控制与两步验证（TOTP）。
- SSRF 出站防护：对 AI/Webhook 等出站请求进行安全守卫。
- 中继模式：网关中继，内网仅一台联网机器代理所有请求到云端。

## 常用命令速查表
- Docker 一键启动（含自动生成密钥）
  - bash <(curl -fsSL https://raw.githubusercontent.com/sreyun/aiops-monitor/master/scripts/secure-compose.sh) && docker compose up -d
- 指定版本启动
  - 编辑 docker-compose.yml，将 :latest 替换为具体版本号后 docker compose up -d
- 二进制运行
  - 服务端：./bin/aiops-server
  - Agent：./bin/aiops-agent --server http://<IP>:8529 --category 生产
- 交叉编译 Agent（Go 1.22+）
  - GOOS=linux   GOARCH=amd64 go build -o bin/aiops-agent-linux-amd64   ./cmd/agent
  - GOOS=linux   GOARCH=arm64 go build -o bin/aiops-agent-linux-arm64   ./cmd/agent
  - GOOS=darwin  GOARCH=amd64 go build -o bin/aiops-agent-darwin-amd64  ./cmd/agent
  - GOOS=darwin  GOARCH=arm64 go build -o bin/aiops-agent-darwin-arm64  ./cmd/agent
  - GOOS=windows GOARCH=amd64 go build -o bin/aiops-agent.exe           ./cmd/agent
- 开机自启
  - Linux（systemd）：cp deploy/aiops-agent.service /etc/systemd/system/ && systemctl enable --now aiops-agent
  - Windows（任务计划）：schtasks /Create /TN "AIOps-Agent" /TR "C:\aiops-agent\start-agent.bat" /SC ONSTART /RU SYSTEM /RL HIGHEST /F
  - macOS（launchd）：cp deploy/com.aiops.agent.plist ~/Library/LaunchAgents/ && launchctl load ~/Library/LaunchAgents/com.aiops.agent.plist

## 故障排查清单
- HTTP 代理“无法解析上游响应”
  - 现象：偶尔正常，常见“unexpected EOF”
  - 根因：session 生命周期管理不当导致竞态；Agent/Server 端连接关闭时机不匹配
  - 修复要点：移除不当全局超时；调整 tx/rx goroutine 关闭逻辑；由 HTTP proxy handler 控制 session 生命周期；改进错误提示
  - 部署步骤：备份旧二进制 → 上传新二进制 → 重启服务与 Agent → 验证
  - 预期效果：不再出现 unexpected EOF；慢服务超时而非挂起；日志包含详细诊断
- 端口转发不可用
  - 检查 forward_listen 与 Docker ports 映射一致
  - 确认规则已启用且未超过最大会话数
  - 查看健康检查接口返回是否 enabled
- 终端无法打开或频繁断开
  - 确认 WebSocket 在反代下正确放行 Upgrade 头
  - 检查二次认证与 MFA 策略
  - 查看会话录制与审计日志
- 采集权限不足（Linux）
  - 现象：部分 /proc 路径被安全模块拦截
  - 处理：临时关闭 SELinux/kysec 宽容模式、以 root 运行、添加白名单
- 告警阈值误报
  - 零值自动兜底默认，避免持续误报
  - 通过面板「告警设置」逐项调整 warn/crit 阈值
- 短信/语音推送失败
  - 核对服务商 provider、AccessKey/SecretKey、app_id、签名与模板 CODE
  - 使用“发送测试”校验连通性

## 相关项目推荐与最佳实践
- 数据存储
  - PostgreSQL：关系数据持久化，支持 pgvector 向量列（RAG 诊断）
  - VictoriaMetrics：时序数据写入与查询
- 安全加固
  - AES-256-GCM 静态加密（AIOPS_SECRET_KEY）
  - TLS 传输加密（AIOPS_TLS_CERT/AIOPS_TLS_KEY）
  - SSRF 出站防护（AIOPS_SSRF_STRICT）
- 高可用与容灾
  - 多服务端推送（单 Agent 广播至多个服务端）
  - 中继模式（跨网络穿透）
- 运维最佳实践
  - 使用固定版本标签与 Compose 管理依赖
  - 合理设置阈值三档（保守/标准/宽松）
  - 开启告警治理（静默/抑制/路由）降低噪音
  - 定期备份 PG 与 VM 数据，制定回滚预案
