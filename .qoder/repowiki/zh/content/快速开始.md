## 简介
AIOps Monitor 提供主机监控、SRE 事件闭环、日志采集检索、AI 巡检诊断等能力。v5.5.0 起统一存储为 PostgreSQL（关系数据）+ VictoriaMetrics（时序数据），并内置首次登录强制安全初始化、AES-256-GCM 静态加密、可选 TLS 传输等安全机制，支持 Docker 一键部署、二进制直接运行、源码编译安装等多种方式，帮助新用户在最短时间内看到监控数据。

## 系统要求与前置依赖
- 操作系统与架构
  - Linux / Windows / macOS；AMD64 与 ARM64 均受支持
- 运行时依赖
  - Agent 侧需要 Python 解释器（Linux/macOS 默认 python3，Windows 默认 python）
- 外部服务（服务端必须）
  - PostgreSQL：承载全部关系数据（用户、配置、审计、事件、工单、会话等）
  - VictoriaMetrics：承载全部时序数据（指标、趋势）
- 端口
  - Web UI/API：8529
  - TCP 转发监听范围：默认 10100-10300（可通过环境变量覆盖）
  - VictoriaMetrics 可选暴露 8428（仅内网建议）

## 从零到运行：三种部署方式

### 方式一：Docker 一键启动（推荐）
- 使用脚本自动下载编排文件并注入强随机密钥，随后一键拉起三容器（aiops-server + postgres + victoriametrics）
- 关键环境变量
  - AIOPS_POSTGRES_DSN：PostgreSQL 连接串（必填）
  - AIOPS_VM_URL：VictoriaMetrics 地址（必填）
  - AIOPS_SECRET_KEY：配置落库主密钥（强烈建议设置）
  - AIOPS_FORWARD_LISTEN：TCP 转发监听地址（Docker 部署需设为 0.0.0.0）
- 浏览器访问 http://localhost:8529，即可看到主机卡片与指标

## 网络与安全初始化
- 网络连通性
  - 确保宿主机或容器能访问 PostgreSQL 与 VictoriaMetrics
  - Docker 部署时，TCP 转发监听地址需设置为 0.0.0.0，以便宿主机或其他容器访问
- 安全初始化流程
  - 首次登录会强制弹出「安全初始化」，必须修改用户名与密码后方可进入
  - 建议随后启用 MFA 两步验证
  - 生产环境务必设置 AIOPS_SECRET_KEY，并对 PG 密码进行替换
- TLS 传输（可选）
  - 通过环境变量提供证书与私钥路径，启用 HTTPS/TLS 加密传输

## 默认凭据与首次登录强制安全初始化
- 默认凭据
  - 用户名：admin
  - 密码：admin
- 首次登录强制安全初始化
  - 检测到默认凭据后，服务端将标记 MustChangePassword，并在登录后返回 must_change_password 标志
  - 用户必须调用账户初始化接口，设置新用户名与新密码，完成后会强制重新登录
- 相关实现要点
  - 登录成功后检测默认凭据并设置强制改密标志
  - 账户初始化接口仅在 MustChangePassword 为真时允许执行，且成功后清除该标志并失效所有会话

## MFA（两步验证）配置
- 功能说明
  - 基于 TOTP（RFC 6238），兼容 Google Authenticator
  - 支持全局策略：管理员可强制要求所有未启用 MFA 的用户在下次登录时完成 MFA 注册
- 操作流程
  - 用户进入 MFA 设置页面，获取二维码与密钥
  - 使用手机认证 App 扫码绑定
  - 提交一次有效的一次性验证码以启用 MFA
  - 管理员可在后台开启全局 MFA 强制策略
- 技术要点
  - 服务器端生成随机 base32 密钥，构造 otpauth URL 与 QR 码
  - 校验时考虑 ±1 时间步容差，防止时钟漂移
  - 全局策略生效时，未启用 MFA 的用户将被限制访问非 MFA 相关接口

## 常见问题与排障
- HTTP 代理“unexpected EOF”问题
  - 现象：HTTP 代理访问偶发失败，错误信息不友好
  - 原因：Agent 端连接目标服务缺乏超时控制，导致挂起或被意外关闭
  - 修复：增加连接与读写超时，改进错误信息与诊断日志
  - 部署：重新编译并替换服务端与 Agent 二进制，重启服务后验证
- 镜像拉取受限
  - 方案 1：配置 Docker 镜像加速器（推荐，AMD64/ARM64 均可用）
  - 方案 2：替换为 SWR 镜像（仅限 AMD64）
- 端口转发不可达
  - 检查 forward_listen 是否为 0.0.0.0（Docker 部署）
  - 确认映射端口范围与防火墙规则一致

## 结论
通过 Docker 一键部署、二进制直接运行或源码编译安装，结合强制安全初始化与可选 MFA、TLS 等安全机制，AIOps Monitor 能在最短时间内帮助新用户成功运行并看到监控数据。生产环境建议采用脚本生成强随机密钥、启用 TLS、合理配置转发监听与端口范围，并根据实际网络拓扑选择是否启用中继模式。
